Big Ideas

Core CS principles and what I’m learning from them

Abstraction

Hide complexity, expose power

From functions and classes to APIs, abstraction lets me build on reliable layers without re-inventing the wheel.

  • Top-down design with clear interfaces
  • Decompose problems into testable units
  • Refactor to remove duplication
Algorithms

Step-by-step solutions

I compare solutions by correctness, readability, and efficiency, using big-O as a guide—not a religion.

  • Trace with sample inputs
  • Measure time/space tradeoffs
  • Prefer clarity, then optimize
Data & Visualization

From numbers to insights

Use the right data types, validate inputs, and visualize trends to support decisions. Test edge cases.

  • Lists, dictionaries/maps, classes
  • Input validation & error handling
  • Charts for exploratory analysis
The Internet

How the web moves bytes

Understanding DNS, HTTP, and Git/GitHub helps me deploy confidently and collaborate with version control.

  • Requests/responses, status codes
  • Static hosting with GitHub Pages
  • Semantic HTML & accessible UI
