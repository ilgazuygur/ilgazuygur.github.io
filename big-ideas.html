Big Ideas

Key notes from AP CSA Unit 1.1–1.9

1.1 – Why Programming?

  • Programming is creating instructions for a computer to solve tasks.
  • Computers execute repetitive and complex problems quickly and accurately.
  • Skills developed: problem decomposition, logical thinking, creativity.
  • Examples: automation, AI, simulations, data analysis, apps.
  • AP CSA builds a strong foundation in Java and problem-solving.

1.2 – Introduction to Java

  • Object-Oriented language: organizes code into classes and objects.
  • Portable: compiles into bytecode that runs on the JVM.
  • Strongly typed: variables need explicit types (e.g., int, String).
  • Basic program structure with class and main method.
  • Case-sensitive; semicolons end statements; braces define blocks.

1.3 – Printing in Java

  • System.out.print(): prints without a new line.
  • System.out.println(): prints and moves to the next line.
  • Concatenation: + joins strings, e.g. "Age: " + 16.
  • Escape sequences: \n (new line), \t (tab), \" (quote).

1.4 – Comments

  • Single-line comments: // text.
  • Multi-line comments: /* text */.
  • Documentation comments for Javadoc: /** ... */.
  • Comments explain purpose, not just what code does. Keep them updated.

1.5 – Variables & Data Types

  • Primitive types (AP scope): int, double, boolean; reference: String.
  • Declare & initialize: int count = 0; double gpa = 3.7; boolean passed = true;
  • Identifiers: start with a letter, no spaces, use camelCase.
  • Constants: final double TAX_RATE = 0.18; (cannot be reassigned).

1.6 – Operators & Expressions

  • Arithmetic: + - * / % (modulo).
  • Integer division truncates: 7/2 ⇒ 3; use casting for real division.
  • Precedence: () → * / % → + -. Use parentheses for clarity.
  • Casting: (double)7/2 ⇒ 3.5; narrowing casts may lose data.
  • Compound: +=, -=, *=, /=, %=.

1.7 – User Input with Scanner

  • import java.util.Scanner; → Scanner in = new Scanner(System.in);
  • Read: in.nextInt(), in.nextDouble(), in.next(), in.nextLine().
  • Newline pitfall: call an extra in.nextLine() after numeric input before reading a full line.
  • Always close when done in larger apps: in.close() (avoid closing System.in if you’ll reuse it).

1.8 – The Math Class

  • Static methods: Math.abs(x), Math.pow(a,b), Math.sqrt(x), Math.max(a,b), Math.min(a,b).
  • Constants: Math.PI, Math.E.
  • Random: Math.random() in [0.0,1.0); example die roll: int die = (int)(Math.random()*6)+1;

1.9 – Errors & Debugging

  • Syntax errors: caught by the compiler (e.g., missing ;).
  • Runtime errors: occur while running (e.g., division by zero, input mismatch).
  • Logic errors: code runs but gives wrong results—use test cases & tracing.
  • Tips: incremental builds, frequent tests, meaningful names, updated comments.
